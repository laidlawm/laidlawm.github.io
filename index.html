Senior Project: Tachyon II FPGA

Project Description

Embedded HPC devices must pass a set of strict requirements to be considered viable. Size, memory, storage, power consumption, and fault tolerance limitations leave little room for performance. This project aims to return performance to these embedded platforms through the use of the RapidIO technology on top of a RISC-V based FPGA. We are working with the company Praesum Communications who are providing the RapidIO Core IP to use with the RISC-V FPGA.

Project Highlights

Using the PolarFire Icicle Kit (a RISC-V based FPGA SoC) as our development board, we've managed bring up a Linux kernel configuration that supports Praesum's RapidIO IP. This same kernel configuration has also been stripped down to remove unessential components. The digital logic of the FPGA SoC has been programmed to implement the Praesum RapidIO Core IP. We've also been able to debug the Praesum RapidIO drivers such that we can interface with the RapidIO subsystem.

My Contributions

  • I was able to configure and test our Linux kernel configuration against the PolarFire Icicle Kit, ensuring its operability on the RISC-V ISA.
  • I was also able to build the RapidIO kernel modules for RISC-V and ensure that they were properly loaded onto the device.
  • Finally, I managed to debug the Praesum RapidIO IP drivers and verify that the OS could observe information on RapidIO devices.
